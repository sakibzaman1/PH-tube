Scope of Var, Let & Const

Scope determines the accessibility of variables.
JavaScript has 3 types of scope:
  • Block scope
  • Function scope
  • Global scope

Var : Variables declared with the var keyword inside a { } block can be accessed from outside the block. Variables declared with the var keyword inside a function are not accessible (visible) from outside the function. Variables declared with the var keyword outside a function can be accessed from anywhere in a JavaScript program.

Let : Variables declared with the let keyword inside a { } block can not be accessed from outside the block. Variables declared with the let keyword inside a function are not accessible (visible) from outside the function. Variables declared with the let keyword outside a function can be accessed from anywhere in a JavaScript program.

Const: Variables declared with the const keyword inside a { } block can not be accessed from outside the block. Variables declared with the const keyword inside a function are not accessible (visible) from outside the function. Variables declared with the const keyword outside a function can be accessed from anywhere in a JavaScript program.

Use case of null & undefined

Null : Null is an object in JavaScript. It is one of JavaScript's primitive values and is treated as falsy for boolean operations. A null value in JavaScript is used for referring intentional absence of any object value and if any function or variable returns null, then we can say that the object could not be created.

Undefined : An undefined variable or anything without a value always returns "undefined" in JavaScript. When a variable is declared and not initialized or not assigned with any value. We will receive undefined when accessing uninitialized variables, non-existing object properties, non-existing array elements, and alike.

REST API

API stands for APPLICATION PROGRAMMING INTERFACE & REST stands for REPRESENTATIONAL STATE TRANSFER.REST API is also called a “RESTful” API. It is a specific type of API. When a client requests a resource using a REST API, the server transfers back the current state of the resource in a standardized representation.RESTful APIs use HTTP requests to interact with data, while traditional APIs can use a variety of protocols.
